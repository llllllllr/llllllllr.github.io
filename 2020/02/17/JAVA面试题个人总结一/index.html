JDK和JRE区别

  • JDK:java development kit，Java开发工具包，提供运行环境和开发环境
  • JRE：java runtime environment java运行工具包，提供java运行环境
  • 只是运行java程序的话就用JRE，需要开发JAVA，用JDK

==和equals的区别

  • 基本类型，都是比较值
  • 引用类型，==比较物理地址，equals重写了之后会比较值

Override equals() and hashCode() for value objects

当在hashMap里面存入KEY时，先用hashCode()方法计算出hash值， 然后把相应的对象放入相应的内存，而默认的hashCode()方法返回的是对象地址。那么，在上述代码中

  • Team k1 = new Team(“New York”, “development”);
  • Team myTeam = new Team(“New York”, “development”);

地址不同，如果根据这个找到相应内存地址，必然什么都没有，所以输出null

这就是只有hashCode()没有equals()的后果

ashMap解决Hash值冲突使用链地址法：

  • hashCode()找到的是一个桶，桶里面装着hashCode()值相同的对象

  • equals()用于在桶里找到确定的对象

  • 综上，hashCode()相等不一定对象相等，反之，equals()返回true，hashCode()返回值必然相等

所以重写equals()同时重写hashCode()，因为

  • 在用HashMap的时候，键（Key）部分，放过自定义对象 的话，要重写hashCode()方法
  • 令equals()返回true的时候，hashCode()必然相等
