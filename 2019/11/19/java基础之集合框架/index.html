常见的set

set的元素是无序的，即存入和取出的顺序不一定一致，元素不可以重复

set的功能和collection是一致的

  • hashset 底层数据结构是哈希表
  • Treeset

set集合的功能和collection的功能是一样的

class person

name:a1 age:5
name:a4 age:8
name:a3 age:7
name:a2 age:6

👆可以看到对象是无序的，因为在内部它是根据哈希值存储的

在class person里面重载

  • public int hashCode()
  • public boolean equals(Object obj)

main 中修改：

a2…a1
a3…a1
a3…a2
a1…a1
name:a1 age:5
name:a2 age:6
name:a3 age:7

可以看到，重复的a1被去掉了。

  • 首先a1进来，哈希值3C
  • a2进来哈希值也是3C，这时调用equals方法，发现equals方法返回false，就让a2顺延下来
  • a3进来分别和a1,a2比较，返回false也顺延
  • a1进来和a1比，equals方法返回true,就不会继续存放了

所以，HashSet是没有重复元素的集合

HashSet的contains
  • 先比较哈希值，相等
  • 进入equals方法，equals返回true，contains返回true

同样的，删除方法也是依赖hashCode()和equals()方法

Map

Hashtable(底层是哈希表数据结构，不允许使用null键和null值，线程同步，效率低)

HashMap(底层是哈希表数据结构，允许使用null键和null值，线程不同步，效率高)

TreeMap（底层是二叉树数据结构，线程不同步，可以用于给map集合中的键进行排序）

Map集合存储键值对，要保证键的唯一性
