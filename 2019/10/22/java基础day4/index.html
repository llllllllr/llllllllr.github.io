  • 避免了单继承的局限性

区别：

  • 实现Runnable线程代码存在接口的子类的run()方法中

多线程的安全问题

比如在卖票程序中👇：

==假如同时有四个线程在运行上面的方法，那么在运行到⭐⭐⭐ ⭐⭐⭐处，依次被挂起，再次激活的时候，不会 不会 不会 再判断 if(ticketNum > 0) 了，直接输出然后 ticketNum-1。==

那么有可能会出现tickNum<0的情况,多线程的运行出现了安全问题

注意：这里面的异常只能在run()方法里面处理，==因为Runnable接口并没有抛出异常==

问题的原因：

  • 当多条语句在操作同一个线程共享数据时，一个线程对多条语句只执行了一部分，还没有执行完，另一个线程参与进来执行，导致共享数据的错误。

解决方法：

  • 对多条操作共享数据的语句，只能让一个线程都执行完，在执行过程中，其它线程不可以参与执行。
  • Java提供==同步代码块==

synchronized(对象){

需要被同步的代码

}

多线程同步代码块

==类似锁，PV操作。==同步，互斥操作。

  • 解决了多线程的安全问题
  • 每次执行前都要判断锁，较为消耗资源

单例设计模式中的线程安全问题

概念：
　　java中单例模式是一种常见的设计模式，单例模式的写法有好几种，这里主要介绍三种：懒汉式单例、饿汉式单例、登记式单例。
　　单例模式有以下特点：
　　1、单例类只能有一个实例。
　　2、单例类必须自己创建自己的唯一实例。
　　3、单例类必须给所有其他对象提供这一实例。
　　单例模式确保某个类只有一个实例，而且自行实例化并向整个系统提供这个实例。在计算机系统中，线程池、缓存、日志对象、对话框、打印机、显卡的驱动程序对象常被设计成单例。这些应用都或多或少具有资源管理器的功能。每台计算机可以有若干个打印机，但只能有一个Printer Spooler，以避免两个打印作业同时输出到打印机中。每台计算机可以有若干通信端口，系统应当集中管理这些通信端口，以避免一个通信端口同时被两个请求同时调用。总之，选择单例模式就是为了避免不一致状态。
