产生原因：

t1 和 t2 负责:

t3 和 t4 负责：

  • 首先是t1获得资格，生产了商品，置flag=true.然后t1唤醒了t2进程（注意，只会唤醒t2线程，即只会唤醒本方法线程）。此时，t1因为已经占有资源，它继续判断，因为flag=true，它搁置，放弃资格，t2进入，搁置，放弃资格
  • t3获得资格，进入，消费，唤醒t4，同理，因为t3已经占有资源，所以它再判断，flag=flase,t4进入，flag=false,无法消费，此时唤醒t1
  • t1生产，唤醒t2.注意，此时t2已经判断过flag标志，它搁置在wait()这个地方，所以它并不会再继续判断flag而是直接生产了！
  • 同理，消费者也是一样，因此会出现上面的问题。

==修改：把if(flag)改成while(flag)==因为这样在wait()之后就会继续判断flag。

Thread-1…生产者…商品—1
Thread-3消费者商品—1
Thread-0…生产者…商品—2
Thread-3消费者商品—2
Thread-1…生产者…商品—3

直接冻结，所有进程都进入等待状态，与死锁有区别
